# Feature: Stock-Flow 专业选股平台升级

**作者**: AI + User  
**日期**: 2026-06-14  
**状态**: Draft

---

## 1. 背景 (Background)

### 1.1 问题描述

当前 stock-flow 是一个「资金流向展示工具」，主要提供大盘资金总览、板块资金流向、个股资金详情等被动查看功能。作为个人投资者的日常分析工具，存在以下核心痛点：

1. **缺乏主动选股能力**：用户每天需要手动浏览各个页面寻找机会，没有系统化的策略推荐
2. **板块追踪碎片化**：概念板块信息散落在不同页面，没有集中的异动追踪和轮动检测
3. **个股信号缺失**：对关注的股票没有每日信号汇总，需要逐个查看技术指标和资金流向
4. **首页信息密度低**：当前"大盘总览"页面偏向数据展示，缺乏决策导向的信息聚合

### 1.2 现状分析

**后端架构**：
- 框架：FastAPI + SQLite + SQLAlchemy
- 数据源：TuShare Pro API（自定义端点 http://124.222.60.121:8020/）
- 缓存：基于 SQLite 的 CacheService，按 trade_date 缓存
- 定时任务：APScheduler，每小时同步数据
- 已有数据表：15 个（MarketFlow, NorthFundFlow, SectorFlow, StockFlow, MoneyflowDc, DragonTiger, LimitList, StockBasic, DailyPrice, SectorMember, IndexDaily, SectorDaily, ThsIndex, StkFactor, DailyBasic）

**后端 API（20+ 端点）**：
- `/api/market/*` — 大盘总览、北向资金、资金趋势、个股排行、涨跌分布、涨跌停、指数
- `/api/sectors/*` — 板块列表、搜索、成分股、趋势
- `/api/stocks/*` — 个股搜索、资金流、龙虎榜、日线、基本面
- `/api/screener/*` — 条件选股（PE/PB/市值/换手率等）
- `/api/technical/*` — 技术指标选股（MACD/KDJ/RSI/布林带等）
- `/api/concept/*` — 概念板块列表、详情、成分股

**前端架构**：
- 框架：React + Ant Design + Vite
- 页面：5 个（大盘总览、板块资金、个股资金、选股器、概念板块）
- 组件：14 个（MarketIndex, FundCard, NorthFundCard, BreadthChart, LimitStats, SectorRanking, StockRanking, FlowTrendChart, TrendChart, TurnoverChart, PriceTrendChart, SectorDetail, StockSearch, StockScreener）
- 已有选股预设：6 个前端硬编码 preset（低估值、高股息、高换手、大市值、小盘股、放量）

**现有选股系统的问题**：
1. 预设策略在前端硬编码，不是后端策略引擎
2. 技术指标选股依赖 stk_factor 预计算值，不能自定义参数
3. 没有策略收益跟踪
4. 没有自选股管理
5. 没有每日信号汇总

### 1.3 主要使用场景

| 场景 | 描述 | 频率 |
|------|------|------|
| 每日开盘前 | 打开应用，查看今日策略推荐、自选股信号、板块异动 | 每天 1 次 |
| 盘中观察 | 追踪关注板块的资金变化，查看个股实时信号 | 每天 2-3 次 |
| 盘后复盘 | 查看策略收益、信号准确率、板块轮动趋势 | 每天 1 次 |
| 周末研究 | 研究新策略、调整自选股组合、回顾历史表现 | 每周 1 次 |

## 2. 目标 (Goals)

> **让个人投资者每天 3 分钟内知道：今天该关注哪些股票、哪些板块在异动、手里的票有什么信号。**

具体目标：
1. **策略驱动选股**：提供 15+ 个专业预设策略，每日自动推荐符合条件的股票
2. **板块异动追踪**：集中展示热点概念、板块轮动趋势、异动预警
3. **个股信号管理**：自选股分组管理 + 每日多维度信号汇总
4. **决策仪表盘**：个性化首页，聚合推荐/信号/异动，一站式决策
5. **AI 工作可见**：记录每次 cron 任务的工作内容，可追溯、可展示

### 模块优先级

| 模块 | 核心功能 | 优先级 |
|------|----------|--------|
| ① 策略推荐中心 | 预设专业策略库 + 每日自动选股 + 策略收益跟踪 | 🔴 P0 |
| ② 板块异动追踪 | 热点概念识别 + 板块轮动检测 + 概念成分股联动 | 🔴 P0 |
| ③ 个股信号系统 | 自选股管理 + 技术/资金/事件信号 + 每日信号汇总 | 🔴 P0 |
| ④ 仪表盘 | 个性化首页 + 信号聚合 + 快捷入口 | 🟡 P1 |
| ⑤ AI 工作日志 | cron 任务执行记录 + SQLite 存储 + 页面展示 | 🔴 P0 |

### 2.1 非目标 (Non-Goals)

| 不做 | 原因 |
|------|------|
| 实时行情推送（WebSocket） | TuShare 不支持实时数据，且个人使用不需要毫秒级推送 |
| 交易下单功能 | 只做分析工具，不涉及券商接口 |
| 多用户/社区功能 | 个人使用，不需要账号体系和分享 |
| 移动端 App | 只做响应式 Web，手机浏览器可用即可 |
| AI 大模型分析 | 先做规则引擎，后续再考虑接入 LLM |
| 策略回测系统 | 本期不做历史回测，只做每日推荐 + 收益跟踪 |
| 自定义策略编辑器 | 本期只做预设策略，用户不能自己写策略逻辑 |

## 3. 需求细化 (Requirements)

### 3.1 功能性需求

#### 模块 ⑤ AI 工作日志（P0）

**FR-5.1 日志记录**
- 每次 cron 任务（代码审查 / 进化 / 数据同步）执行后，自动记录工作内容
- 记录字段：任务类型、执行时间、耗时、执行结果摘要、修改的文件列表、是否成功
- 任务类型：`review`（代码审查）、`evolution`（进化）、`sync`（数据同步）、`manual`（手动触发）

**FR-5.2 日志存储**
- 新增 SQLite 表 `activity_log`：
  - `id` INTEGER PRIMARY KEY
  - `task_type` TEXT NOT NULL — review/evolution/sync/manual
  - `task_name` TEXT — 任务名称（如"stock-flow 激进进化"）
  - `started_at` TEXT NOT NULL — 开始时间
  - `finished_at` TEXT — 结束时间
  - `duration_seconds` INTEGER — 耗时秒数
  - `status` TEXT NOT NULL — success/failed/partial
  - `summary` TEXT — 执行结果摘要（AI 生成的一段话）
  - `files_changed` TEXT — 修改的文件列表（JSON 数组）
  - `details` TEXT — 详细报告（完整 markdown）
  - `created_at` TEXT DEFAULT CURRENT_TIMESTAMP

**FR-5.3 日志写入机制**
- cron 任务执行完毕后，由 Hermes agent 将执行结果通过 API 写入
- 提供 `POST /api/activity-log` 接口供 cron 任务回调
- 接口接收 JSON：`{ task_type, task_name, summary, files_changed, details, status }`
- 自动填充 `started_at`、`finished_at`、`duration_seconds`

**FR-5.4 日志展示页面**
- 新增「AI 日志」页面，侧边栏新增入口（图标：RobotOutlined）
- 日志列表：按时间倒序，每条显示任务类型标签、任务名、时间、耗时、状态
- 点击展开：显示完整执行报告（markdown 渲染）
- 支持按任务类型筛选（全部/审查/进化/同步）
- 支持按时间范围筛选（今天/本周/本月/全部）

**FR-5.5 仪表盘集成**
- 仪表盘新增「最近活动」卡片
- 显示最近 5 条 AI 工作日志
- 每条显示：任务类型、摘要、时间
- 点击跳转到日志详情页

**FR-5.6 日志 API**
- `POST /api/activity-log` — 写入日志（供 cron 任务调用）
- `GET /api/activity-log` — 日志列表（支持分页、筛选）
- `GET /api/activity-log/{id}` — 日志详情
- `GET /api/activity-log/stats` — 统计摘要（本周做了多少次审查、进化了多少次、修改了多少文件）

#### 模块 ① 策略推荐中心（P0）

**FR-1.1 策略引擎架构**
- 后端实现统一的 `StrategyEngine`，每个策略是一个 `Strategy` 子类
- 策略注册机制：新策略只需继承基类、实现 `evaluate()` 方法即可注册
- 每个策略包含：名称、描述、类别、评估逻辑、参数配置
- 策略引擎支持按类别筛选、按胜率排序

**FR-1.2 预设策略库（15+ 策略）**

| 类别 | 策略名 | 数据源 | 核心逻辑 |
|------|--------|--------|----------|
| 💎 价值 | 低估值金矿 | daily_basic | PE(TTM)<15 + PB<2 + 股息率>3% + 总市值>50亿 |
| 💎 价值 | 高股息防守 | daily_basic | 股息率 TOP50 + PE<20 |
| 💎 价值 | 破净有资金 | daily_basic + moneyflow_dc | PB<1 + 近3日净流入>0 |
| 🚀 动量 | 放量突破 | daily + daily_basic | 量>5日均量×2 + 涨幅>3% + 换手率 3%-15% |
| 🚀 动量 | 均线多头排列 | daily (计算MA) | MA5>MA10>MA20>MA60 + 当日收阳 |
| 🚀 动量 | MACD金叉 | stk_factor | DIF上穿DEA + 柱状线由负转正 |
| 🚀 动量 | KDJ超卖反弹 | stk_factor | K值<20 + K上穿D |
| 💰 资金 | 主力持续流入 | moneyflow_dc | 连续3日主力净流入>0 + 累计TOP30 |
| 💰 资金 | 北向资金加仓 | hk_hold | 近5日北向增持 + 个股涨幅<3% |
| 💰 资金 | 融资余额增长 | margin | 融资余额连续3日增长>5% |
| ⚡ 事件 | 龙虎榜机构抢筹 | dragon_tiger | 机构席位净买入>1亿 |
| ⚡ 事件 | 大宗交易溢价 | block_trade | 大宗交易成交价>收盘价 |
| ⚡ 事件 | 连板强势股 | limit_list_d | 连续涨停≥2天 + 开板次数<3 |
| 🔗 组合 | 价值+资金共振 | daily_basic + moneyflow_dc | PE<20 + 近3日净流入>0 + 换手率>2% |
| 🔗 组合 | 趋势+量能共振 | daily + stk_factor | MA5>MA10 + 量比>1.5 + 涨幅>1% |
| 🔗 组合 | 超跌反弹 | daily + stk_factor | 近10日跌幅>15% + RSI(14)<30 + 当日止跌 |

**FR-1.3 策略推荐结果**
- 每个策略返回符合条件的股票列表（最多 30 只）
- 每只股票显示：名称、代码、行业、收盘价、涨跌幅、关键指标值
- 支持按涨跌幅、市值、换手率等排序
- 推荐结果按 trade_date 缓存，避免重复计算

**FR-1.4 策略收益跟踪**
- 记录每个策略每日推荐的股票列表
- 追踪推荐后 1日/3日/5日 收益率（从 daily 表获取）
- 计算策略胜率（正收益比例）、平均收益率
- 前端展示：策略卡片上显示近N日胜率、平均收益

**FR-1.5 策略 API**
- `GET /api/strategies` — 策略列表（含类别、描述、胜率）
- `GET /api/strategies/{strategy_id}/recommend` — 获取策略推荐结果
- `GET /api/strategies/{strategy_id}/performance` — 获取策略收益统计

#### 模块 ② 板块异动追踪（P0）

**FR-2.1 热点概念识别**
- 每日涨幅 TOP10 概念板块排行
- 每个概念显示：名称、涨幅、资金净流入、成分股数量、领涨股
- 点击概念 → 跳转到概念详情页（已有 `/api/concept/{ts_code}`）

**FR-2.2 板块轮动检测**
- 追踪概念板块连续 N 日（3/5/10日）资金净流入趋势
- 识别「持续流入」板块（连续3日净流入）和「突然流入」板块（当日大幅流入）
- 前端展示：板块轮动趋势图（类似热力图的时间序列）

**FR-2.3 概念成分股联动**
- 概念详情页增加「资金流向」tab
- 显示成分股的资金流向排行
- 支持按净流入排序

**FR-2.4 板块异动预警**
- 当概念板块单日涨幅 > 3% 或资金净流入 > 阈值时标记为「异动」
- 异动板块在仪表盘置顶显示
- 异动原因自动标注（如：政策利好、业绩超预期等，基于 Tushare 事件数据）

**FR-2.5 板块 API**
- `GET /api/concept/hot` — 热点概念排行（TOP10）
- `GET /api/concept/rotation` — 板块轮动趋势（3/5/10日）
- `GET /api/concept/alerts` — 板块异动列表

#### 模块 ③ 个股信号系统（P0）

**FR-3.1 自选股管理**
- 前端：自选股列表页面，支持添加/删除/分组
- 分组：观察仓、重仓、长线（预设分组，可自定义名称）
- 数据存储：SQLite 表 `watchlist`（id, ts_code, group_name, added_date, notes）
- 每个自选股显示：名称、代码、最新价、涨跌幅、当日信号数

**FR-3.2 每日信号汇总**
- 对每个自选股，当日产生的所有信号汇总
- 信号类型：
  - 技术信号：MACD金叉/死叉、KDJ超买超卖、RSI超买超卖、布林带突破
  - 资金信号：主力大幅流入（>5000万）、主力大幅流出、北向资金增持/减持
  - 事件信号：涨停/跌停、龙虎榜上榜、大宗交易
- 每个信号包含：信号类型、强度（强/中/弱）、描述、触发时间

**FR-3.3 信号强度计算**
- 强信号：多维度共振（如 MACD金叉 + 主力流入 + 量比放大）
- 中信号：单维度信号（如 仅 MACD金叉）
- 弱信号：参考级信号（如 RSI 接近超卖区但未触发）

**FR-3.4 信号 API**
- `GET /api/watchlist` — 自选股列表
- `POST /api/watchlist` — 添加自选股
- `DELETE /api/watchlist/{ts_code}` — 删除自选股
- `GET /api/watchlist/{ts_code}/signals` — 获取个股当日信号
- `GET /api/watchlist/signals/summary` — 获取所有自选股信号汇总

#### 模块 ④ 仪表盘（P1）

**FR-4.1 个性化首页**
- 替代当前的「大盘总览」作为默认首页
- 仪表盘由多个卡片组成，可折叠/展开

**FR-4.2 仪表盘卡片**
- **策略推荐卡片**：今日推荐 TOP5 股票（来自策略引擎）
- **板块热力图**：概念板块涨跌热力图（绿色=跌，红色=涨）
- **自选股信号流**：所有自选股的今日信号，按时间倒序
- **大盘概览**：三大指数 + 资金总览 + 涨跌分布（精简版）
- **板块异动**：今日异动板块列表

**FR-4.3 仪表盘 API**
- `GET /api/dashboard` — 聚合所有仪表盘数据（一次请求）

### 3.2 非功能性需求

| 类别 | 要求 | 说明 |
|------|------|------|
| 性能 | 仪表盘加载 < 3s | 策略计算可异步，仪表盘读缓存 |
| 性能 | 策略推荐 API < 5s | 复杂策略（如连续3日资金流入）可能较慢 |
| 可用性 | 策略计算失败不阻塞 | 单个策略失败不影响其他策略和页面 |
| 数据一致性 | 推荐结果按日缓存 | 同一 trade_date 的推荐结果不变 |
| 可扩展 | 新增策略 < 30 分钟 | 策略注册机制，继承基类实现 evaluate() |
| 可维护 | 策略逻辑与 API 解耦 | 策略引擎独立于路由层 |
| 兼容性 | 不破坏现有 API | 所有现有端点保持不变 |
| 资源 | 单次策略计算内存 < 500MB | 全市场约 5000 只股票 |
