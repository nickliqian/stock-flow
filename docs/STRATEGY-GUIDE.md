## 1. 概述：系统设计理念

### 1.1 设计哲学

Stock Flow 的核心理念是**多因子共振**——单一策略在特定市场环境下有效，但没有任何策略能适应所有行情。系统通过以下设计实现稳健性：

```
┌────────────────────────────────────────────────────────────────┐
│                    策略有效性 = f(市场状态, 因子组合, 时间周期)      │
└────────────────────────────────────────────────────────────────┘
```

**三大原则：**

1. **因子分散**：覆盖价值、动量、资金、事件、组合 5 大因子类别，避免因子拥挤
2. **信号验证**：每个策略信号需经过多维度验证（技术面、资金面、基本面）
3. **动态适应**：策略权重随市场状态自动调整，识别策略衰减并触发进化

### 1.2 系统架构

## 3. 策略详细说明

### 3.1 MACD 金叉 (macd_golden_cross)

**分类**: 动量 | **图标**: 📈

#### 背景
MACD（Moving Average Convergence Divergence）是最经典的趋势跟踪指标之一。金叉信号表示短期动能超越长期动能，是趋势启动的早期信号。

#### 算法逻辑

```
DIF = EMA(close, 12) - EMA(close, 26)
DEA = EMA(DIF, 9)
MACD = (DIF - DEA) × 2
```

#### 信号规则

| 条件 | 阈值 | 权重 |
|------|------|------|
| DIF > DEA | — | 必须 |
| MACD > 0 | — | 必须 |
| pct_change > 0 | — | 必须 |
| close > 2 | 过滤低价股 | 必须 |
| DIF > 0 | 零轴上方 | +30 分 |
| MACD 强度 | max(40) | 0-40 分 |
| 涨幅大小 | max(30) | 0-30 分 |

#### 评分公式

```python
macd_score = min(max(macd / 0.5, 0), 40)      # MACD 柱状线强度
pct_score = min(pct_change * 3, 30)            # 涨幅贡献
zero_bonus = 30 if dif > 0 else 0              # 零轴上方加分
total_score = min(macd_score + pct_score + zero_bonus, 100)
```

#### 适用场景
- **最佳**: 趋势启动初期、震荡市突破
- **较差**: 单边下跌、极度缩量
- **持有周期**: 3-10 个交易日

### 3.2 均线多头排列 (ma_alignment)

**分类**: 动量 | **图标**: 📊

#### 背景
均线多头排列是经典的趋势确认信号。当短期均线全部位于长期均线之上，且按时间周期从小到大排列时，表明市场处于强势上涨趋势。

#### 信号规则

```
MA5 > MA10 > MA20 > MA60
```

#### 评分维度

| 条件 | 说明 |
|------|------|
| 排列完整度 | 满足几层均线关系 |
| 乖离率 | 短期均线与长期均线的距离 |
| 趋势斜率 | 均线上行的角度 |
| 成交量配合 | 放量确认 |

#### 适用场景
- **最佳**: 趋势行情、突破后的确认
- **较差**: 震荡市（频繁假信号）
- **持有周期**: 5-20 个交易日

---

### 3.3 KDJ 超卖反弹 (kdj_oversold_rebound)

**分类**: 动量 | **图标**: 🔄

#### 背景
KDJ 是超买超卖指标，当 K/D 值进入超卖区（<20）后金叉，往往预示短期反弹机会。

#### 算法逻辑

```
RSV = (close - low_9) / (high_9 - low_9) × 100
K = 2/3 × K(-1) + 1/3 × RSV
D = 2/3 × D(-1) + 1/3 × K
J = 3K - 2D
```

#### 信号规则

| 条件 | 阈值 |
|------|------|
| K < 20 | 超卖区 |
| D < 20 | 超卖区 |
| K > D | 金叉 |
| J < 0 | 极度超卖（加分） |

#### 适用场景
- **最佳**: 震荡市的短期反弹
- **较差**: 单边下跌（可能钝化）
- **持有周期**: 1-5 个交易日

### 3.4 趋势量价共振 (trend_volume_resonance)

**分类**: 动量 | **图标**: 📈

#### 背景
量价配合是技术分析的核心原则。价升量增表明资金流入，趋势可信度高；价升量缩则可能是假突破。

#### 信号规则

```
条件组合：
1. close > open（阳线）
2. volume > volume_ma20（放量）
3. pct_change > 1%（有涨幅）
4. 趋势向上（MA5 > MA10）
```

#### 评分维度

| 因子 | 权重 | 计算方式 |
|------|------|---------|
| 量比 | 30% | volume / volume_ma20 |
| 涨幅 | 30% | pct_change |
| 趋势强度 | 40% | MA5 斜率 |

#### 适用场景
- **最佳**: 趋势行情的确认信号
- **较差**: 顶部放量（可能是出货）
- **持有周期**: 3-10 个交易日

### 3.5 放量突破 (volume_breakthrough)

**分类**: 动量 | **图标**: 🚀

#### 背景
成交量突破是价格突破的确认信号。当成交量超过 20 日均量的 1.5 倍以上，且价格创出近期新高，往往预示新一轮上涨。

#### 信号规则

```
条件组合：
1. volume > 1.5 × volume_ma20（放量）
2. close > max(close_20)（突破 20 日高点）
3. pct_change > 2%（有力度）
```

#### 评分维度

| 因子 | 权重 |
|------|------|
| 量比 | 40% |
| 突破幅度 | 30% |
| 涨幅 | 30% |

#### 适用场景
- **最佳**: 震荡市结束后的突破
- **较差**: 连续大涨后的高位突破
- **持有周期**: 5-15 个交易日

### 3.6 低估值金矿 (low_valuation_gold)

**分类**: 价值 | **图标**: 💎

#### 背景
低估值策略基于价值投资理念，寻找 PE、PB 均低于行业平均的股票，等待估值修复。

#### 信号规则

```
条件组合：
1. pe_ttm < 15（PE 低于 15）
2. pb < 1.5（PB 低于 1.5）
3. total_mv > 50 亿（排除小盘股）
4. turnover_rate > 0.5%（保证流动性）
```

#### 评分维度

| 因子 | 权重 | 计算方式 |
|------|------|---------|
| PE 低估程度 | 40% | 15 / pe_ttm |
| PB 低估程度 | 30% | 1.5 / pb |
| 市值适中 | 20% | 50-500 亿最优 |
| 股息率 | 10% | dv_ttm |

#### 适用场景
- **最佳**: 熊市末期、震荡市
- **较差**: 牛市（可能跑输成长股）
- **持有周期**: 20-60 个交易日

### 3.7 高股息策略 (high_dividend)

**分类**: 价值 | **图标**: 💰

#### 背景
高股息策略适合稳健型投资者，通过持有高分红股票获取稳定现金流，同时享受估值修复收益。

#### 信号规则

```
条件组合：
1. dv_ttm > 3%（股息率 > 3%）
2. pe_ttm < 20（估值合理）
3. 连续 3 年分红（分红持续性）
4. total_mv > 100 亿（蓝筹优先）
```

#### 评分维度

| 因子 | 权重 |
|------|------|
| 股息率 | 50% |
| 估值水平 | 30% |
| 分红持续性 | 20% |

#### 适用场景
- **最佳**: 震荡市、防御配置
- **较差**: 牛市初期（弹性不足）
- **持有周期**: 60+ 个交易日

### 3.8 主力资金流入 (main_fund_inflow)

**分类**: 资金 | **图标**: 💵

#### 背景
主力资金（大单）的流入流出是股价走势的重要领先指标。当主力资金持续流入，往往预示股价上涨。

#### 信号规则

```
条件组合：
1. net_amount > 0（净流入）
2. buy_lg_amount > sell_lg_amount（大单买入 > 卖出）
3. 连续 2 日净流入
```

#### 评分维度

| 因子 | 权重 | 计算方式 |
|------|------|---------|
| 净流入额 | 40% | net_amount / total_mv |
| 大单占比 | 30% | buy_lg / (buy_lg + sell_lg) |
| 连续性 | 30% | 连续流入天数 |

#### 适用场景
- **最佳**: 趋势行情的确认信号
- **较差**: 震荡市（主力反复）
- **持有周期**: 3-10 个交易日

### 3.9 涨停回封 (limit_up_reseal)

**分类**: 事件 | **图标**: 🎯

#### 背景
涨停后打开又回封，说明多头力量强劲，是强势股的典型特征。

#### 信号规则

```
条件组合：
1. 当日涨停（pct_change >= 9.9%）
2. 盘中打开涨停（最高价 > 涨停价）
3. 收盘回封涨停
4. 换手率 > 5%（充分换手）
```

#### 评分维度

| 因子 | 权重 |
|------|------|
| 回封力度 | 40% |
| 换手率 | 30% |
| 封单量 | 30% |

#### 适用场景
- **最佳**: 强势股的二次确认
- **较差**: 连续涨停后（风险高）
- **持有周期**: 1-3 个交易日

### 3.10 断层金坑 (broken_net_gold)

**分类**: 事件 | **图标**: 🕳️

#### 背景
跳空低开后企稳，形成"断层"，往往是恐慌性抛售后的底部信号。

#### 信号规则

```
条件组合：
1. 跳空低开（open < 昨收 × 0.97）
2. 当日收阳或十字星
3. 成交量放大（恐慌抛售）
4. 下影线 > 上影线（下探企稳）
```

#### 适用场景
- **最佳**: 急跌后的反弹
- **较差**: 趋势性下跌（可能是中继）
- **持有周期**: 3-10 个交易日

### 3.11 板块轮动 (sector_rotation)

**分类**: 组合 | **图标**: 🔄

#### 背景
板块轮动是 A 股市场的典型特征。通过追踪板块资金流向变化，可以提前布局轮动方向。

#### 算法逻辑

```
1. 计算各板块近 N 日资金净流入
2. 计算资金流入加速度（二阶导数）
3. 识别启动期板块（加速流入）
4. 识别衰退期板块（加速流出）
```

#### 信号规则

| 阶段 | 特征 | 操作 |
|------|------|------|
| 启动期 | 资金流入加速 | 买入 |
| 持续期 | 资金流入平稳 | 持有 |
| 衰退期 | 资金流出加速 | 卖出 |

#### 适用场景
- **最佳**: 震荡市、结构行情
- **较差**: 全面牛市（所有板块同涨）
- **持有周期**: 5-20 个交易日

### 3.12 多周期动量 (multi_timeframe_momentum)

**分类**: 动量 | **图标**: 📊

#### 背景
多周期共振是趋势交易的核心理念。当日线、周线、月线同时发出买入信号时，趋势的可靠性最高。

#### 信号规则

```
日线：MA5 > MA10（短期趋势向上）
周线：MA5 > MA10（中期趋势向上）
月线：MA5 > MA10（长期趋势向上）

三周期共振 = 信号强度最高
两周期共振 = 信号强度中等
单周期信号 = 信号强度最低
```

#### 评分维度

| 因子 | 权重 |
|------|------|
| 日线信号 | 30% |
| 周线信号 | 40% |
| 月线信号 | 30% |

#### 适用场景
- **最佳**: 大趋势行情
- **较差**: 震荡市（信号稀少）
- **持有周期**: 20-60 个交易日

## 4. 高级分析引擎

### 4.1 资金流向背离分析 (Flow Intelligence)

**API**: `/api/strategies/flow-intelligence/divergence-scan`

#### 核心概念
资金流向背离是指价格与资金流向出现相反走势的现象。常见的背离类型：

| 背离类型 | 价格 | 资金流 | 含义 |
|---------|------|--------|------|
| 看涨背离 | 下跌 | 流入 | 主力吸筹 |
| 看跌背离 | 上涨 | 流出 | 主力出货 |
| 量价背离 | 上涨 | 缩量 | 趋势衰竭 |

#### 算法逻辑

```python
# 计算价格趋势
price_trend = calculate_trend(close, window=10)

# 计算资金流趋势
flow_trend = calculate_trend(net_amount, window=10)

# 检测背离
if price_trend < 0 and flow_trend > 0:
    signal = "bullish_divergence"  # 看涨背离
elif price_trend > 0 and flow_trend < 0:
    signal = "bearish_divergence"  # 看跌背离

# 计算背离强度
strength = abs(flow_trend) / abs(price_trend) * 100
```

#### 信号解读

| 信号 | 强度 | 操作建议 |
|------|------|---------|
| 看涨背离 | >70 | 关注买入机会 |
| 看涨背离 | 50-70 | 观望 |
| 看跌背离 | >70 | 考虑减仓 |
| 看跌背离 | 50-70 | 提高警惕 |

### 4.3 市场宽度与温度指标 (Market Breadth)

**API**: `/api/market-breadth/`

#### 核心概念
市场宽度衡量上涨/下跌股票的比例，反映市场的整体健康状况。

#### 关键指标

| 指标 | 公式 | 解读 |
|------|------|------|
| 涨跌比 | 上涨家数 / 下跌家数 | >1 为多头市场 |
| 新高新低比 | 创新高 / 创新低 | >1 为强势 |
| 均线突破比 | 站上 MA20 / 跌破 MA20 | >0.5 为多头 |
| 市场温度 | 综合得分 (0-100) | >70 过热，<30 过冷 |

#### 温度指标

```
温度 = 0.3 × 涨跌比得分 + 0.3 × 新高新低比得分 + 0.4 × 均线突破比得分

温度区间：
- 0-20: 极冷（恐慌抛售）
- 20-40: 偏冷（底部区域）
- 40-60: 中性（震荡市）
- 60-80: 偏热（上涨趋势）
- 80-100: 极热（可能见顶）
```

### 4.4 波动率聚类与风险分区 (Volatility Clustering)

**API**: `/api/volatility/`

#### 核心概念
波动率聚类分析市场的波动特征，识别高波动和低波动区域，为仓位管理提供依据。

#### 算法逻辑

```python
# 计算历史波动率
hist_vol = std(returns, window=20) * sqrt(252)

# 计算波动率百分位
percentile = percentile_rank(hist_vol, lookback=252)

# 风险分区
if percentile > 80:
    zone = "high_volatility"   # 高波动区，降低仓位
elif percentile < 20:
    zone = "low_volatility"    # 低波动区，可能变盘
else:
    zone = "normal"            # 正常波动
```

#### 风险分区建议

| 分区 | 波动率百分位 | 仓位建议 |
|------|-------------|---------|
| 高波动 | >80% | 50% 以下 |
| 正常 | 20-80% | 70-80% |
| 低波动 | <20% | 80-100%（注意变盘） |

**API**: `/api/strategies/evolution/*`

#### 核心概念
策略会随时间衰减，自进化引擎通过持续监控策略表现，检测衰减信号并触发优化。

#### 生命周期模型

```
        胜率
         │
    100% ┤      ╭─────╮
         │     ╱       ╲
     80% ┤    ╱  成熟期  ╲
         │   ╱           ╲
     60% ┤  ╱  成长期    ╲ 衰退期
         │ ╱               ╲
     40% ┤╱  导入期          ╲
         │                    ╲
     20% ┤                     ╲
         │
      0% ┼───────────────────────── 时间
         0   30  60  90  120  150  180
```

#### 衰减检测算法

```python
# 计算近期胜率
recent_winrate = count(wins[-20:]) / 20

# 计算历史胜率
hist_winrate = count(wins[-60:]) / 60

# 衰减分数
decay_score = (hist_winrate - recent_winrate) / hist_winrate

if decay_score > 0.3:
    status = "decaying"    # 明显衰减
elif decay_score > 0.15:
    status = "warning"     # 轻微衰减
else:
    status = "healthy"     # 健康
```

### 4.6 协整配对交易 (Pair Trading)

**API**: `/api/pair-trading/*`

#### 核心概念
配对交易基于均值回归思想，寻找两只高度相关的股票，当价差偏离均值时做多低估股、做空高估股。

#### 算法逻辑

```python
# 1. 计算股票对的相关性
correlation = pearsonr(stock_a.close, stock_b.close)

# 2. 协整检验
coint_test = coint(stock_a.close, stock_b.close)
is_cointegrated = coint_test.pvalue < 0.05

# 3. 计算价差
spread = stock_a.close - beta * stock_b.close

# 4. 计算 Z-Score
z_score = (spread - spread_mean) / spread_std

# 5. 信号规则
if z_score > 2:      # 做空 A，做多 B
elif z_score < -2:   # 做多 A，做空 B
```

#### 适用场景
- **最佳**: 震荡市、同行业股票
- **较差**: 趋势市、非相关股票
- **持有周期**: 5-15 个交易日

### 4.7 因子轮动模型 (Factor Model)

**API**: `/api/strategies/factor-model/*`

#### 核心概念
因子轮动基于"没有永远有效的因子"这一理念，通过追踪各因子的动态表现，轮动配置当前最有效的因子。

#### 因子分类

| 因子 | 代表策略 | 相关指标 |
|------|---------|---------|
| 价值 | 低估值金矿、高股息 | PE, PB, 股息率 |
| 动量 | MACD 金叉、均线多头 | 收盘价、均线 |
| 资金 | 主力流入、智能追踪 | 净流入额、大单占比 |
| 质量 | 股东增持、内部人信心 | 增持比例、质押率 |

#### 轮动算法

```python
# 1. 计算各因子近期收益
factor_returns = {
    "value": calculate_factor_return("value", recent_days=5),
    "momentum": calculate_factor_return("momentum", recent_days=5),
    "flow": calculate_factor_return("flow", recent_days=5),
}

# 2. 计算因子动量
factor_momentum = {
    k: recent_return - historical_return
    for k, recent_return in factor_returns.items()
}

# 3. 选择动量最强的因子
top_factors = sorted(factor_momentum.items(), key=lambda x: -x[1])[:2]

# 4. 基于选中因子选股
picks = select_stocks_by_factors(top_factors)
```

### 4.9 策略拥挤度演进 (Crowding Evolution)

**API**: `/api/strategies/crowding-*`

#### 核心概念
当太多投资者使用相同策略时，策略的有效性会下降（拥挤效应）。拥挤度演进追踪各策略的使用热度，预警拥挤风险。

#### 拥挤度指标

| 指标 | 计算方式 | 解读 |
|------|---------|------|
| 选股宽度 | 被选中股票数 / 总股票数 | 越宽越健康 |
| 拥挤比率 | 重叠股票数 / 选中股票数 | 越高越拥挤 |
| 多样性指数 | 活跃策略数 / 总策略数 | 越高越分散 |

#### 拥挤度告警

```
告警级别：
- 🟢 正常: 拥挤度 < 30%
- 🟡 警告: 拥挤度 30-50%
- 🔴 危险: 拥挤度 > 50%

操作建议：
- 正常: 正常使用策略
- 警告: 关注策略衰减信号
- 危险: 考虑切换策略或降低仓位
```

## 6. 组合构建方法论

### 6.1 策略组合原则

1. **因子分散**: 不同类别的策略组合
2. **低相关性**: 选择收益相关性低的策略
3. **风险预算**: 每个策略分配固定风险预算
4. **动态调整**: 根据市场状态调整权重

### 6.2 自适应权重算法

```python
# 基于近期表现动态调整权重
def calculate_adaptive_weights(strategies, lookback_days=20):
    weights = {}
    for strategy in strategies:
        # 计算近期夏普比率
        sharpe = calculate_sharpe(strategy.returns[-lookback_days:])

        # 计算胜率
        winrate = strategy.wins[-lookback_days:] / lookback_days

        # 综合得分
        score = 0.5 * sharpe + 0.5 * winrate

        weights[strategy.name] = score

    # 归一化
    total = sum(weights.values())
    return {k: v / total for k, v in weights.items()}
```

### 6.3 组合构建流程

```
1. 策略选择
   ├── 因子类别覆盖
   ├── 历史表现筛选
   └── 低相关性验证

2. 权重分配
   ├── 等权重（默认）
   ├── 风险平价
   └── 自适应权重

3. 选股执行
   ├── 各策略独立选股
   ├── 信号矩阵聚合
   └── 去重与评分

4. 仓位管理
   ├── 单股上限（如 5%）
   ├── 行业上限（如 20%）
   └── 总仓位控制
```
